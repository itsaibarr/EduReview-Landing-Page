# Brand Book — Education Engagement Platform

> Studio-level standard. No AI patterns. Every decision is intentional.

---

## 1. Design Philosophy

The product sits at the intersection of institutional trust and human motivation.
The design must communicate both — not by being loud, but by being precise.

**Three words that govern every decision:**
- **Clarity** — nothing decorative that doesn't serve meaning
- **Restraint** — whitespace is not empty space, it is structure
- **Confidence** — the design does not beg for attention, it earns it

**What this is NOT:**
- Gradient mesh blobs floating everywhere
- Generic SaaS purple-to-blue Aurora orbs
- CSS `background-clip: text` gradient headlines (overused, looks AI-generated)
- Every element animated for the sake of motion
- Anything that looks like it was generated by an AI design tool

**Visual DNA (synthesis of references, not copies):**
- **CoreShift**: Floating island navbar, connected node graph visual, massive clean headline, pure white background — the floating elements ARE the visual
- **Reel.ai / dark SaaS**: Metric badge cards as floating data proof, social proof logo strip
- **Linear**: Typographic precision, controlled motion
- **Stripe**: Information hierarchy, spacing discipline

## 2. Typography

### Primary Font: Satoshi

- Source: [Fontshare](https://www.fontshare.com/fonts/satoshi) — free, commercial-friendly, variable font
- Designer: Deni Anggara
- Character: Swiss modernist. Geometric structure with warm rounded terminals. Not cold, not casual.

### Monospace: JetBrains Mono

- Use only for data labels, metric values, and code snippets
- Never for body copy or UI text

---

### Typographic Rules

- Display headlines break at intentional points — never let the browser decide the line break
- **No CSS gradient text** — this is an AI-pattern tell. Instead, one single word or short phrase uses solid `#2563EB` (brand blue) inline. This is editorial, not decorative.
  - Example: `See who's` **`learning.`** ← "learning" is `text-brand`, rest is `text-primary`
- Body copy max-width: `560px` — never stretches full-width
- No centered body copy beyond 2 lines — always left-align paragraphs
- Headings centered in hero, left-aligned in all other sections

### Gradient Definitions

```css
/* CTA button — subtle, not neon */
--gradient-cta: linear-gradient(135deg, #2563EB 0%, #3B82F6 100%);

/* Section background wash — sections only, never hero */
--gradient-section: linear-gradient(180deg, #EFF6FF 0%, #FFFFFF 100%);
```

**Hero background: pure white (`#FFFFFF`).**
No gradient, no wash, no effect. The floating node graph provides all visual interest.
Adding a background effect on top of floating elements is noise, not signal.
The whitespace IS the design. Trust it.

## 7. Animation System

### Principles

- **Motion serves meaning.** If removing an animation doesn't reduce comprehension, remove it.
- **Entrances, not exits.** Elements animate in; they do not animate out on scroll.
- **One hero, controlled everywhere else.** The hero has the strongest motion. Sections use subtle reveals only.
- **No attention-seeking.** Nothing bounces, spins, or pulses unless it communicates status.

### Timing Functions

```css
--ease-out: cubic-bezier(0.16, 1, 0.3, 1);        /* Primary — entrances */
--ease-in-out: cubic-bezier(0.45, 0, 0.55, 1);    /* Background motion */
--ease-spring: cubic-bezier(0.34, 1.56, 0.64, 1); /* Hover micro — very subtle */
```

### Duration Scale

| Use | Duration |
|---|---|
| Micro (hover color, border) | `150ms` |
| Component (button scale, card lift) | `200ms` |
| Entrance (fadeInUp) | `500–600ms` |
| Node card float (hero) | `4000–7000ms` infinite, alternating |
| Connector line draw | `1200ms` on mount |

### Scroll Reveal (Global)

All non-hero sections: `y: 20 → 0`, `opacity: 0 → 1`, `duration: 500ms`, `ease: --ease-out`.
Stagger between siblings: `60ms`.
Trigger: `whileInView`, `once: true`, `margin: -80px`.

### Hero Node Graph Visual System

The hero visual is a **connected node graph** — floating metric cards linked by thin dashed SVG lines with dot markers at intersections. Inspired by CoreShift's integration graph, adapted with our data language.

**Composition:**
- 4–5 floating metric cards positioned above the headline text
- Connected by thin SVG `<line>` elements: `stroke: #BFDBFE`, `strokeWidth: 1`, `strokeDasharray: "5 5"`
- Dot markers at intersections: `r: 3`, `fill: #DBEAFE`, `stroke: #BFDBFE`
- Cards float with slow, gentle vertical oscillation — different phase offsets per card so they feel organic, not synchronized

**Cards (Institution path):**
| Card | Content | Size |
|---|---|---|
| Engagement score | `87 / 100` + label "Engagement Index" | Medium |
| Growth indicator | `+12%` + label "This month" | Small |
| At-risk flag | `3 students` + "Need attention" | Small |
| Attendance | `94%` + bar visual | Medium |

**Cards (Student path):**
| Card | Content | Size |
|---|---|---|
| Progress | `+18 pts` + label "vs last month" | Medium |
| Streak | `14 days` + streak icon | Small |
| Subject | `Math ↑` + trend arrow | Small |

**Card style:**
- Background: `#FFFFFF`
- Border: `1px solid #E0E7FF`
- Shadow: `--shadow-sm`
- Radius: `--radius-lg` (16px)
- Padding: `12px 16px`
- Data value: `JetBrains Mono 600 18px #0A0A14`
- Label: `Satoshi 400 12px #8B92AD`
- Accent dot: `8px × 8px`, `border-radius: 50%`, `#2563EB` or semantic color

**Float animation per card:**
```
animate={{ y: [0, -8, 0] }}
transition={{ duration: 5, repeat: Infinity, ease: "easeInOut", delay: cardIndex * 1.2 }}
```

**Connector lines:**
Drawn as absolute-positioned SVG overlaid on the card cluster. Coordinates are fixed to the card positions. No physics — deliberately static connections that suggest relationship, not chaos.

## 8. Component Rules

### Buttons

**Primary — Hero CTA (pill, standalone):**
- Background: `--gradient-cta`
- Text: `#FFFFFF`, `Satoshi 600`, `15px`
- Padding: `13px 28px`
- Radius: `--radius-full` (pill) — hero CTA is the ONE place pills are used deliberately
- Hover: `translateY(-1px)` + `--shadow-brand`
- Active: `translateY(0)` + `scale(0.99)`
- Transition: `200ms --ease-out`

**Primary — Navbar CTA (compact):**
- Same background and text
- Padding: `9px 18px`
- Radius: `--radius-md` (10px) — tighter context, more contained feel
- Same hover behavior

**Secondary (ghost):**
- Background: `transparent`
- Border: `1px solid --border` (`#E0E7FF`)
- Text: `--text-secondary`
- Hover: border → `--brand-200`, text → `--brand`

**Text link:**
- No border, no background
- Color: `--brand`
- Hover: `--brand-700` with `→` nudging right `2px`

### Badges / Pills

- Background: `--brand-100`
- Text: `--brand-700`, `Satoshi 500`, `13px`
- Padding: `4px 12px`
- Radius: `--radius-full`
- No border — background color provides boundary

### Cards

- Background: `--white`
- Border: `1px solid --border-subtle`
- Shadow: `--shadow-sm`
- Radius: `--radius-lg`
- Hover: `--shadow-md` + `translateY(-2px)`, `300ms --ease-out`
- No bright colored borders — blue borders are for active/focus states only

---

### Navbar — Floating Island

The navbar is a **contained, centered floating bar** — not edge-to-edge.
It is a distinct object on the page, not a stripe across the top.

```
position: fixed
top: 24px
left: 50%
transform: translateX(-50%)
width: fit-content
min-width: 640px
max-width: 780px

background: rgba(255,255,255,0.92)
backdrop-filter: blur(16px)
border: 1px solid #E0E7FF
border-radius: 16px
padding: 8px 8px 8px 20px
shadow: 0 4px 24px rgba(0,0,0,0.06), 0 1px 4px rgba(0,0,0,0.04)
```

Logo is left-anchored within the bar. Nav links are in the center. CTA is right-anchored with slightly tighter padding than hero CTA. This creates a visually balanced, premium "command bar" feel.

On scroll > 80px: `border-color` shifts to `#BFDBFE`, shadow intensifies slightly.

---

### Social Proof Strip

Immediately below hero CTAs. Signals institutional validation.

```
margin-top: 48px
display: flex, align-items: center, gap: 32px
opacity: 0.45
filter: grayscale(100%)
transition: opacity 200ms — hover brings to 0.65
```

Labels like "Currently in pilot with:" in `Satoshi 400 13px #8B92AD` followed by institution name marks or placeholder text. Keep minimal — 3–4 items max.
